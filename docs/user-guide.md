# Watchtower (瞭望塔) 用户手册

---

## 1. 快速开始

### 1.1 环境要求

- **Go** 1.21+
- **Node.js** 18+（仅开发前端需要）
- **操作系统**：macOS / Linux（推荐）
- **数据库**：无需外部数据库（内嵌 SQLite）

### 1.2 启动方式

```bash
# 1. 克隆项目
git clone <repo>
cd watchtower

# 2. 配置（可选）
# 编辑 configs/config.yaml 修改端口、ES 等配置
# 编辑 configs/.env 设置密码和密钥

# 3. 构建前端（如修改了 frontend/ 下的源码）
cd frontend && npm install && npm run build && cd ..

# 4. 编译运行
go build -o scm .
./scm

# 5. 访问
# 浏览器打开 http://localhost:3972
# 默认账号密码：admin / admin
```

### 1.3 配置说明

| 配置文件 | 路径 | 说明 |
|---------|------|------|
| 主配置 | `configs/config.yaml` | 端口（默认 3972）、ES 地址、探针间隔（默认 15s） |
| 环境变量 | `configs/.env` | `ES_PASSWORD`、`AUTH_JWT_SECRET`、`ADMIN_USER`、`ADMIN_PASSWORD_HASH` |

> `.env` 文件会在启动时自动加载，无需手动 `source`。

## 2. 功能模块详解

### 2.1 服务大盘（Dashboard）

服务大盘是应用首页，集中展示所有主机的运行状态。

**顶部统计卡片：**
- 总主机数
- 健康主机数
- 异常主机数
- 维护中主机数
- 角色异常数

**主机列表：**
- 每台主机以卡片形式展示
- **LED 指示灯**：绿 = 存活（ICMP），红 = 离线（ICMP）
- **主机详情**：名称、IP、CPU、内存、磁盘用量
- **存活状态标签**：`存活: 在线 / 离线 / 未知` — 基于 ICMP ping 探测
- **服务角色标签**：如 `nginx: 正常 / 异常 / 探测中` — 基于角色独立探针（含错误信息悬浮提示）
- 两个标签分开显示，互不混淆

**操作按钮：**
- **角色筛选按钮**：按服务类型过滤主机列表
- **刷新按钮**：手动触发一次拨测，立即更新所有主机的存活状态
- **导出 Excel**：将当前数据导出为单 sheet Excel 文件（含 CPU(核)、内存(GB) 单位，磁盘合并显示）

### 2.2 主机管理

**添加主机：**
- 点击"添加主机"按钮
- 填写：名称、IP、CPU、内存、磁盘分区（支持多分区）
- 磁盘分区格式：`挂载点:大小`，可通过表格添加多行
  - 示例：`/` → 200（GB），`/data` → 500（GB）
- 添加成功后自动绑定 ICMP 存活探测

**编辑主机：**
- 点击主机卡片上的编辑按钮
- 可修改：名称、IP、CPU、内存、磁盘分区
- **角色变更**：可在编辑表单中选择新角色
  - 选择角色保存后，系统自动**删除旧角色分配**并**创建新角色分配**
  - 保存后自动触发探针刷新，2 秒后更新状态

**删除主机：**
- 点击主机卡片上的删除按钮，确认后移除

**维护模式：**
- 切换主机为维护状态
- 维护期间暂停对该主机的所有探测

**批量添加：**
- 支持一次粘贴多行 IP 批量导入
- 适用于快速初始化大批量主机

### 2.3 角色管理

**添加角色：**
- **名称**：角色显示名称
- **探测类型**：ICMP / TCP / HTTP / SSH
  - 选择类型后**自动填充默认端口**：HTTP→80、SSH→22
- **端口**：非 ICMP 类型必填
- **检测路径**：仅 HTTP 类型需要
- **超时**：探测超时时间（秒，默认 5）

**删除角色：**
- 可删除自定义创建的角色
- 系统预置角色（ICMP 等）也可删除

**批量创建：**
- 支持一次提交创建多个角色

> **ICMP 角色特殊处理**：ICMP 角色仅用于存活探测，不会显示在主机卡片上的服务标签中。

### 2.4 角色分配

为每个主机分配服务角色，系统将根据角色类型对主机执行定期探测。

- 在主机编辑表单的角色下拉框中选择
- 选择并保存后自动完成分配
- 角色变更时自动删除旧分配并创建新分配
- 分配后触发一次探测，主机卡片 2 秒后更新角色状态
- **角色移除**：编辑表单中选择"不分配角色"即可解除绑定

### 2.5 SSH 终端

通过浏览器直接 SSH 连接目标主机，使用 xterm.js 终端模拟器。

**操作步骤：**
1. 点击主机卡片上的 **SSH 按钮**（主机存活时可用）
2. 弹出凭据选择对话框：
   - **使用已保存的凭据**：从凭据列表单选
   - **手动输入**：输入用户名 + 密码或私钥
3. 凭据连接成功后，终端窗口直接显示，不显示用户名/密码输入框
4. 手动连接成功后，可看到用户名/密码输入框和连接/断开按钮

**凭据管理：**
- 添加 SSH 凭据：标签、用户名、密码或私钥
- 删除 SSH 凭据
- 凭据保存后，密码字段脱敏显示为 `******`

### 2.6 架构图

架构图功能基于 **Vue Flow** 库实现，用于可视化展示主机与服务的拓扑关系。

**操作入口：**
- 切换到"架构图"标签页

**四种交互模式：**

| 模式 | 操作方式 | 用途 |
|------|---------|------|
| 选择模式 | 拖拽移动节点/文字/分组框；点击可选中 | 布局调整、选中元素 |
| 连线模式 | 点击源节点连接点 → 点击目标节点连接点 | 创建节点间的连线 |
| 文字模式 | 点击画布空白处 | 添加文字标注 |
| 分组模式 | 在画布上拖拽画矩形框 | 将一组服务框选为逻辑分组 |

**节点连接：**
- 每个服务节点有 **8 个连接点**：左上、上中、右上、右中、右下、下中、左下、左中
- 连接点默认隐藏，鼠标悬停时显示
- 连线模式下所有连接点常显

**连线样式：**
- 支持 smoothstep/bezier/straight/step 四种路线类型
- 支持实线/虚线切换
- 支持箭头/无箭头切换

**分组框：**
- 可调整大小
- 可选择填充颜色和边框颜色
- 可添加名称标识（如"生产环境"、"测试环境"）
- 支持虚线和实线样式

**数据持久化：**
- **保存 / 恢复**：数据保存在浏览器 localStorage 中（key=`scm-diagram-data`）
- **导出 PDF**：一键导出为 A4 横向 PDF（纯白背景，不含控件和网格）
- **一键清除**：清空画布所有内容

### 2.7 日志监控

日志监控用于实时查看日志流并配置告警规则。

**四个子标签：**

| 子标签 | 功能说明 |
|--------|---------|
| 告警规则 | 管理日志告警规则，支持关键词匹配、正则匹配、排除关键词、冷却时间、指定 Webhook 等配置 |
| ES 配置 | 配置 Elasticsearch 连接信息（地址、用户名、密码、索引、轮询间隔、每次查询最大日志数），显示连接状态 |
| 实时日志 | 通过 WebSocket 实时推送日志流，显示时间戳、日志级别(INFO/ERROR)、告警令牌剩余等信息 |
| 限流缓存 | 查看被限流策略拦截的告警记录 |

**告警规则：**
- 规则包含：名称、关键词、排除关键词、级别过滤、正则表达式、冷却时间、告警模板、关联 Webhook
- 支持启用/停用切换

**ES 配置（Web UI）：**
- 地址、用户名、密码、索引（如 `k8s-logs-*`）
- 轮询间隔（秒）
- 每次查询最大日志数
- 启用/停用日志轮询
- 连接状态指示器

**实时日志显示：**
- 时间戳：`2026/07/11 09:01:59` 格式
- 日志级别：彩色标签（INFO=蓝色、WARN=黄色、ERROR=红色）
- 告警匹配时显示规则名称
- 告警成功显示"告警成功"，告警失败显示错误信息
- 显示剩余令牌数

**告警推送：**
- 支持 **飞书** / **钉钉** / **企业微信** Webhook 告警推送
- 支持多个 Webhook 独立管理（添加/删除/编辑）
- Webhook 支持限流配置（每分钟 / 每秒发送上限）
- 支持 @提及（所有人/指定人员/不 @）
- 支持自定义消息模板（含 `{rule_name}`、`{level}`、`{message}` 等变量）
- 支持模板测试
- 告警去重：基于 MD5 签名 + 5 分钟 TTL，避免重复告警

### 2.8 Excel 导出

**导出格式：** `.xlsx`（使用 Excelize 库生成）

**Sheet 结构：**

| Sheet 名称 | 内容 |
|-----------|------|
| 主机列表 | 基础信息：名称、地址、状态（中文）、**CPU(核)**、**内存(GB)**、磁盘（挂载点+容量）、角色、维护模式 |

> 不再使用分区独立 Sheet，所有磁盘信息合并显示在主机列表行的磁盘列中。

## 3. 常见操作

### 3.1 添加一台服务器

1. 点击 **"添加主机"**
2. 填写主机名和 IP 地址
3. 可选填：CPU、内存、磁盘分区（可添加多行）
4. 点击 **确定**
5. 系统自动开始 ICMP 存活探测
6. 等待数秒，LED 指示灯变绿即表示主机存活

### 3.2 监控一个服务（如 Nginx）

1. 点击 **"添加角色"**，选择类型 **HTTP**，端口自动填充 **80**
2. 在主机编辑表单中，从角色下拉框选择刚创建的 Nginx 角色
3. 点击 **确定** 保存
4. 系统开始定时探测 Nginx 服务的连通性
5. 主机卡片显示两个标签：
   - `存活: 在线` — 主机 ICMP 在线
   - `nginx: 正常` — Nginx 服务 80 端口可达

### 3.3 变更主机角色

1. 点击主机卡片的 **"编辑"**
2. 在角色下拉框中选择新角色（或选择"不分配角色"解除绑定）
3. 点击 **确定**
4. 系统自动 **删除旧角色分配**，**创建新角色分配**，并立即触发探测
5. 2 秒后主机卡片自动刷新，显示新角色的探测状态

### 3.4 绘制架构图

1. 切换到 **"架构图"** 标签
2. 从左侧主机列表拖拽主机到画布（服务框自动显示其角色名）
3. 切换到 **"连线模式"**，点击连接点创建连线
4. 使用 **"文字模式"** 添加标注
5. 使用 **"分组模式"** 框选分组（可调整大小、颜色、名称）
6. 点击 **"保存"** 存储到浏览器
7. 点击 **"PDF"** 导出为 A4 横向 PDF（纯白背景，无控件和网格）

### 3.5 通过凭据 SSH 连接

1. 先添加 SSH 凭据（设置 → 凭据管理）
2. 点击主机卡片的 **SSH** 按钮
3. 在凭据选择弹窗中选择已保存的凭据
4. 点击 **"确定"**
5. 终端自动连接，不显示用户名/密码输入框

### 3.6 配置日志告警

1. 切换到 **"日志监控" → "ES 配置"**
2. 填写 ES 地址、用户名、密码、索引、轮询间隔、每次查询最大日志数
3. 点击 **保存**，确认连接状态为"已连接"
4. 切换到 **"告警规则"**，添加规则（如关键词 `ERROR`）
5. （可选）在 **"Webhook 配置"** 中添加飞书/钉钉/企业微信 Webhook
6. 在告警规则中指定使用的 Webhook
7. 当 ES 中有匹配的日志时，自动推送告警并显示在实时日志中

## 4. 注意事项

- **首次启动**：系统自动创建管理员账号（默认 `admin` / `admin`），请及时修改密码。
- **ICMP 探测**：需要 `ping` 命令可用。程序会自动查找 `/sbin/ping`、`/usr/sbin/ping` 等常见路径，无需手动配置。
- **SSH 终端**：需要目标主机已配置并开启 SSH 服务。
- **日志监控**：需要在 Web UI 中配置 Elasticsearch 连接信息（或通过 `configs/config.yaml` + `configs/.env` 配置）。
- **架构图数据**：存储在浏览器 localStorage 中，不同设备之间不共享，清除浏览器数据会导致架构图丢失，请及时导出 PDF 备份。
- **前端开发**：如需修改前端代码，需在 `frontend/` 目录下开发（Vue 3 + Vite），执行 `npm run build` 构建后重启后端生效。
- **告警令牌**：Webhook 限流令牌每分钟/每秒刷新，实时日志中可查看剩余令牌数。
